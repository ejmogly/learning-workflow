## 2. What I Learned
This section captures the core concepts I learned from this topic.

- Databricks SQL provides an environment for querying, transforming, and analyzing large-scale data within the Databricks ecosystem.
- It is not just a SQL editor, but part of a broader workflow that includes tables, notebooks, Delta Lake, and collaborative analytics engineering practices.
- Databricks SQL is especially useful for building analytics-ready datasets, validating transformations, and supporting BI/reporting use cases.
- Delta tables are important because they improve reliability through structured table storage and support more manageable downstream workflows.
- Working in Databricks also means thinking about schemas, catalogs, table organization, and how SQL outputs connect to reusable data assets.
- Key takeaway: Databricks SQL is valuable not only for writing queries, but for building a scalable and organized analytics environment where transformation logic can be reused and governed more effectively.
