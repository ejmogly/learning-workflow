## 2. What I Learned
This section captures the core concepts I learned from this topic.

- Well-structured SQL is not just for querying data, but for building reusable and reliable transformation logic.
- SQL for Data Analytics Engineering should focus on transformation, aggregation, validation, readability, and mart-oriented design.
- CTEs, window functions, filtering logic, and validation queries are essential building blocks for analytics-ready datasets.
- Key takeaway: SQL should be written not only to answer one question, but also to support repeatable downstream analytics and reliable data usage.

This section explains why this topic is important in real-world work.

### Operational Value
